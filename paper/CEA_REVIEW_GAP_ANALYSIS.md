# 《计算机工程与应用》YOLO 类论文对标与差距分析

本文档用于指导期刊长文重写。它不是正文引用源，也不直接提供实验数值；其中的论文题录需要在 CNKI、万方、期刊官网或学校数据库中逐条核验后，才能进入正式参考文献。

## 1. 对标结论

从《计算机工程与应用》及相近中文科技期刊的 YOLO 改进论文看，审稿更关注以下几点：

1. 问题场景必须具体，例如无人机航拍、小目标、密集遮挡、复杂背景、轻量部署。
2. 方法不能只写“加入模块”，需要说明每个模块对应解决哪个问题。
3. 实验必须覆盖 baseline、主流模型对比、消融实验、复杂度、速度和可视化。
4. 小目标论文最好有尺度分布、类别级表现或困难样例分析。
5. 结论要强调精度、速度、参数量之间的折中，而不是简单宣称全面最优。
6. 参考文献要覆盖近三年 YOLO 系列、无人机检测、小目标检测、多尺度融合和注意力机制。

## 3. 期刊化写作框架

### 3.1 引言应回答的问题

- 为什么无人机航拍目标检测不同于普通目标检测。
- 为什么 VisDrone 中 small 目标比例高会导致 YOLO nano 模型漏检。
- 为什么只增加模型容量不是唯一解，还需要考虑轻量化和速度。
- 为什么 P2 分支、高分辨率输入和位置敏感注意力适合这个问题。
- 本文贡献必须与真实实验对应，不提前承诺未完成对照。

建议贡献写法：

1. 构建轻量化 HRPCA-YOLO11n，将 P2 高分辨率检测分支与 CoordAttention 结合，用于增强无人机小目标的浅层空间信息和位置敏感表达。
2. 设计同协议实验矩阵，比较 YOLO11n、YOLOv8n、YOLO11s 及高分辨率对照，分析结构、分辨率和模型容量对 VisDrone 检测性能的影响。
3. 基于真实标注和预测结果开展尺度分布与尺度分组匹配分析，说明改进模型的收益主要集中在 small 和 medium 目标。
4. 从类别级指标、速度复杂度和可视化失败案例讨论模型的适用范围与局限性。

### 3.2 方法部分应避免的问题

- 避免只写“加入 P2 后提升小目标检测”，要说明 P2 引入的是更高空间分辨率特征。
- 避免把 CoordAttention 写成主要性能来源，当前真实结果显示其增益有限。
- 避免把 960 输入写成结构创新，应写为高分辨率输入策略和实验变量。
- 避免把 SmallObjAug 写成成功方法，当前结果只能作为有价值的负向/边界消融。

## 5. 当前稿件距离中刊投稿的主要不足

1. 论文篇幅仍偏短，方法和相关工作论证不够像期刊长文。
2. 当前最强结果中 960 输入贡献很大，需要公平分辨率对照完成后再定结论。
3. 与近年中文 YOLO 无人机论文相比，参考文献数量和对比讨论仍不足。
4. 方法结构图偏简略，需要做成正式框图。
5. 外部对比还不能混用不同输入分辨率直接下结论，需要分层讨论。
6. 缺少一张清晰的精度-速度-参数量折中图。
7. 失败案例分析还需要从“展示图片”升级为“归因讨论”。
