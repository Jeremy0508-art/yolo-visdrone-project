# 《计算机工程与应用》投稿元信息工作表

本文档用于在正式投稿前收集和核对投稿系统、期刊模板、导师审阅和论文首页可能需要的元信息。它不是官方模板，也不自动证明论文可以上传；所有字段应由作者和导师在最终投稿前人工确认。

## 1. 稿件基本信息

| 字段 | 待填写内容 | 核对说明 |
| --- | --- | --- |
| 中文题名 | 待填写 | 建议突出 VisDrone、无人机航拍、小目标检测、轻量化或高分辨率 P2 分支中的核心主线 |
| 英文题名 | 待填写 | 与中文题名含义一致，避免夸大“state-of-the-art”或“universal superiority” |
| 投稿栏目 | 待填写 | 根据期刊最新栏目选择，如模式识别与人工智能、图形图像处理、工程与应用等 |
| 稿件类型 | 待填写 | 研究论文、应用研究、工程应用或期刊系统可选类型 |
| 当前 PDF | `paper/manuscript_submission_candidate.pdf` | 投稿前需按官方模板重新核对 |
| 当前 LaTeX 源 | `paper/manuscript_submission_candidate.tex` | 若期刊要求 Word，需要人工迁移 |
| 当前主要数据集 | VisDrone2019-DET | 当前论文只报告验证集结果，不报告官方 test-dev AP |

## 2. 作者与单位信息

| 字段 | 待填写内容 | 核对说明 |
| --- | --- | --- |
| 作者顺序 | 待填写 | 必须由导师/课题组确认 |
| 第一作者 | 待填写 | 与投稿系统、论文首页、作者简介保持一致 |
| 通信作者 | 待填写 | 确认是否需要星号、脚注、邮箱和电话 |
| 作者单位中文 | 待填写 | 按学校/学院/实验室正式名称填写 |
| 作者单位英文 | 待填写 | 与学校官方英文名称一致 |
| 通信地址 | 待填写 | 若投稿系统要求，填写单位地址和邮编 |
| 作者邮箱 | 待填写 | 至少通信作者邮箱必须可用 |
| 作者电话 | 待填写 | 若投稿系统要求，填写可联系手机号或办公电话 |
| ORCID | 待填写 | 仅在系统要求或作者已有时填写 |
| 作者简介 | 待填写 | 若期刊模板要求，按模板格式补充研究方向、职称或学位信息 |

## 3. 摘要与关键词核对

| 字段 | 待填写内容 | 核对说明 |
| --- | --- | --- |
| 中文摘要最终版 | 待确认 | 应包含目的、方法、结果、结论；结果数值必须来自 `paper/tables/` |
| 英文摘要最终版 | 待确认 | 与中文摘要一致，不新增中文摘要没有的结论 |
| 中文关键词 | 待确认 | 建议 3-5 个，术语与正文一致 |
| 英文关键词 | 待确认 | 与中文关键词对应 |
| 摘要中是否包含验证集边界 | 待确认 | 当前只能写 VisDrone 验证集结果 |
| 摘要中是否避免过度结论 | 待确认 | 不写“全面优于所有 YOLO 模型” |

## 使用规则

- 不要在本表中填写未经导师确认的作者、基金或声明信息。
- 不要把第三方网页的模板、费用或周期信息当作官方要求。
- 不要在官方 test-dev 平台返回结果前填写 test-dev AP。
- 若正式投稿格式与当前 LaTeX PDF 不一致，应先更新 `paper/CEA_TEMPLATE_MIGRATION_RECORD.md`，再迁移正文和图表。
