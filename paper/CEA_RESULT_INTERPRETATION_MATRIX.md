# 《计算机工程与应用》结果判读矩阵

本文档用于服务器公平对照实验完成后的论文改写决策。它不新增实验数值，也不预设结论；它规定当真实结果出现不同走向时，论文应该如何调整主张、图表和文字表述。

## 1. 使用原则

- 只使用完整 100 epoch 训练结果、同步后的 `results.csv`、`args.yaml`、权重文件和审计通过的 `paper/tables/`。
- 半程训练指标只能作为进度记录，不能进入本文档的判断结论。
- 文献中的指标可以用于相关工作叙述，不能与本项目复现实验混入同一公平对照表。
- 如果真实结果削弱当前方法主张，应如实改写论文，而不是隐藏负向结果。
- 摘要和结论最后更新，必须由最终结果表支撑。

## 3. 结果分支与论文主张

### 情况 A：主方法明显优于 YOLO11n-960

可采用主张：

> 在相同 960 输入分辨率下，P2 高分辨率检测分支和 CoordAttention 仍能带来额外增益，说明本文方法的提升不仅来自输入分辨率增加，也与结构设计有关。

对应改写：

- 摘要可强调“同分辨率下仍取得提升”。
- 方法部分重点解释 P2 与 CoordAttention 对小目标细节和位置信息的作用。
- 实验部分增加同分辨率公平对照表。
- 结论可写“结构改进在 960 输入下仍具有有效性”。

注意：

- 仍需报告速度、参数量和 GFLOPs 成本。
- 不写“全面优于所有 YOLO 模型”，除非外部基线也支持。

### 情况 B：主方法与 YOLO11n-960 接近

可采用主张：

> 高分辨率输入是 VisDrone 小目标检测性能提升的重要来源，P2 与 CoordAttention 更多体现为轻量结构补充和小目标场景适配。

对应改写：

- 摘要弱化“结构显著提升”，改为“系统分析分辨率、结构和速度折中”。
- 方法部分强调设计动机，而不是绝对增益。
- 实验讨论重点转向分辨率收益、速度成本和结构补充价值。
- 结论避免把 P2/CA 写成主要贡献。

注意：

- 这仍可以形成期刊论文，但类型更偏系统实证分析。
- 小目标尺度分析、速度复杂度分析和失败案例需要写得更充分。

### 情况 C：YOLO11n-P2-960 优于 YOLO11n-960，但 CA 增益有限

可采用主张：

> P2 高分辨率检测分支是主要结构贡献，CoordAttention 的作用相对有限或受训练设置影响。

对应改写：

- 方法标题和贡献列表中突出 P2 分支。
- CoordAttention 作为辅助模块讨论，不夸大其贡献。
- 消融表中如实呈现 CA 的增益或不足。
- 讨论部分说明注意力模块在密集小目标场景中的适用边界。

### 情况 D：YOLO11s-960 精度高于主方法

可采用主张：

> 更大容量模型提供了更高精度上限，本文方法的价值需要从轻量化、参数效率、速度和部署成本角度讨论。

对应改写：

- 主结果表中将 YOLO11s-960 标为容量参考模型。
- 精度-速度-复杂度图必须作为重点图。
- 结论只在参数效率或速度证据支持时强调轻量化优势。
- 如果速度或参数效率也不占优，则改写为局限性和后续优化方向。

注意：

- 不能把低容量模型输给高容量模型解释为失败；关键是说明不同部署目标下的取舍。

### 情况 E：YOLOv8n-960 与主方法接近或更优

可采用主张：

> 不同 YOLO 版本在相同数据和分辨率协议下可能表现接近，本文结果主要说明 YOLO11n 体系内结构与分辨率设计的影响。

对应改写：

- 外部基线章节单独讨论，不与内部消融混在一起。
- 摘要避免“优于主流 YOLO”这类泛化表述。
- 相关工作中补充 YOLOv8 系列在航拍小目标任务中的竞争力。
- 结论限定在本实验协议下。

### 情况 F：SmallObjAug 表现不如结构改进

可采用主张：

> 简单小目标友好增强策略并不必然带来提升，过强或不匹配的数据增强可能破坏密集目标场景的有效上下文。

对应改写：

- 将 SmallObjAug 保留为负向消融。
- 讨论增强策略与结构改进的差异。
- 不把该实验写成正向创新点。

## 4. 表格与图表更新规则

| 材料 | 更新触发条件 | 要求 |
| --- | --- | --- |
| 主结果表 | 任一公平实验完整同步后 | 只加入审计通过的完整 run |
| 消融表 | 同族模型结果改变后 | 区分 640 消融与 960 公平对照 |
| 复杂度表 | 新模型权重可用后 | 参数量、GFLOPs、权重大小来自真实脚本输出 |
| 速度表 | 新模型进入论文比较后 | 同一硬件和同一脚本协议下测量 |
| 精度-速度图 | 速度或复杂度表更新后 | 不绘制缺失速度的模型点 |
| 尺度分组表 | 最终比较模型集合改变后 | 明确它是阈值匹配分析，不等同官方 AP |
| 可视化图 | 最终主模型确定后 | 选择能体现密集、小尺度、遮挡和失败场景的样例 |

## 5. 摘要与结论写作门槛

摘要可以更新的条件：

- 主结果表已包含最终纳入论文的公平对照模型。
- 速度和复杂度表已覆盖摘要中提到的模型。
- 所有提升幅度均能追溯到 `paper/tables/`。

结论可以更新的条件：

- 公平分辨率对照已经完成或明确不纳入。
- 外部基线对照已经完成或明确不纳入。
- 负向结果已经在讨论或局限性中如实处理。
- PDF 编译和审计通过。

## 6. 禁止表述

在公平实验完全同步和判读前，不使用以下表述：

- “全面优于所有 YOLO 模型”
- “显著优于主流方法”
- “官方 VisDrone test-dev 结果表明”
- “所有类别均获得提升”
- “无需额外计算成本”

只有在真实表格、速度结果、类别级结果或官方反馈支持时，才能使用相应结论。
