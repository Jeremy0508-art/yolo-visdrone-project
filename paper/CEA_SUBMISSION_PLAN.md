# 《计算机工程与应用》投稿强化计划

## 目标定位

本计划的目标是把当前 YOLO-VisDrone 项目推进到《计算机工程与应用》投稿准备状态。录用结果受审稿专家、选题热度和同期稿件影响，无法保证 100% 录用；本项目可控目标是把论文质量、实验完整度、证据链、格式规范和可复现材料做到接近期刊投稿要求。

期刊侧要求以学术论文标准格式、结构严谨完整、层次清晰为基本要求，一般论文建议 7500 字以上。项目后续所有论文稿件应按这一标准扩展，而不是保持会议短文体量。

参考来源：

- 《计算机工程与应用》投稿须知：`https://jsjgcyyy.juqk.net/buy/`
- 《计算机工程与应用》编辑部页面：`https://jsjgcyyy.juqk.net/`

## 与《计算机工程与应用》同类 YOLO 论文的差距

《计算机工程与应用》同类 YOLO 改进论文通常强调以下证据链：

1. 明确任务痛点，例如无人机小目标、密集遮挡、长尾类别、复杂背景。
2. 提出可命名的改进方法，而不是简单堆叠已有模块。
3. 至少包含主流模型对比、改进前后对比、消融实验、复杂度和速度分析。
4. 给出可视化对比和失败原因分析。
5. 参考文献覆盖近三年 YOLO、小目标检测、无人机检测相关工作。
6. 论文篇幅和论证深度达到期刊而不是会议短文水平。

当前主要差距：

| 差距 | 风险 | 补强动作 |
| --- | --- | --- |
| 创新点表述偏弱 | 审稿人可能认为只是 P2、CA、960 的组合 | 将方法重构为“轻量高分辨率多尺度感知网络”，明确问题-模块-证据对应关系 |
| 960 输入贡献需要清晰解释 | YOLO11n-960、YOLO11n-P2-960、YOLOv8n-960、YOLO11s-960 均已完成，结果表明分辨率和模型容量都很关键 | 在论文讨论中区分高分辨率输入、P2 分支和模型容量贡献 |
| 外部对比主张需收敛 | YOLO11s-960 精度高于本文轻量模型，不能写成全面领先 | 将外部基线用于说明容量上限和轻量折中，而不是写绝对优势 |
| 小目标专项指标不足 | “小目标检测”主题支撑不够 | 新增按目标面积分组的 AP/Recall 统计 |
| 论文篇幅不足 | 不符合期刊论证深度 | 扩写引言、相关工作、方法细节、实验分析和局限性 |
| 格式未完全对齐期刊模板 | 投稿前返工风险 | 后续迁移到期刊 Word/LaTeX 格式，并准备图表源文件 |

## 核心论文主张

后续论文不应简单宣称“本文方法全面优于所有 YOLO 模型”，而应采用更稳健的主张：

> 面向 VisDrone 无人机航拍小目标检测，本文构建一种轻量高分辨率多尺度检测方法，通过 P2 高分辨率分支、位置敏感注意力和高分辨率输入增强小目标特征表达，并系统分析模型结构、输入分辨率和模型容量对精度、速度与复杂度的影响。

建议暂定方法名称：

> HRPCA-YOLO11n：High-Resolution P2 and Coordinate Attention YOLO11n

中文名称：

> 融合高分辨率 P2 分支与坐标注意力的轻量化 YOLO11n 无人机小目标检测方法

该名称后续可根据实验结果调整。

## 小目标专项分析

必须新增 `tools/analyze_object_scales.py` 或等价脚本，基于验证集标注和预测结果统计：

1. 验证集 GT 目标面积分布。
2. small / medium / large 分组数量占比。
3. 按面积分组的 Precision、Recall、mAP50 或至少 Recall/匹配率。
4. 各类别小目标比例。
5. 小目标失败案例图。

建议面积分组：

| 分组 | 定义 |
| --- | --- |
| small | bbox 面积小于 32×32 像素，或相对图像面积低于阈值 |
| medium | 介于 small 和 large 之间 |
| large | bbox 面积大于 96×96 像素，或相对图像面积较高 |

最终输出：

- `paper/tables/object_scale_distribution.csv`
- `paper/tables/scale_group_results.csv`
- `paper/figures/scale_analysis/`
- 论文新增“小目标尺度分析”小节。

## 证据审计规则

1. 所有论文数值必须记录来源文件。
2. 每个实验必须有：
   - config
   - command
   - run directory
   - results.csv
   - args.yaml
   - best.pt metadata
   - log file
3. 未完成实验不得进入论文表格。
4. 官方 test-dev 结果只有在官方平台返回后才能写入论文。
5. 任何从论文或网页摘录的文献数据，必须标明不是同协议复现结果。

- 新建本计划文档。

优先顺序：

- 扩展 `tools/export_paper_tables.py` 支持所有新增实验。

### Phase 4：论文全面重写

- 以《计算机工程与应用》期刊论文标准扩写。

- 引言与相关工作必须强调近三年 YOLO、无人机、小目标检测进展。

### Phase 5：投稿材料准备

- 生成 PDF 预览。

## 风险与应对

| 风险 | 应对 |
| --- | --- |
| YOLO11s-960 超过本文模型 | 改写主张为“轻量模型在参数效率上更优”，或将 YOLO11s 作为上限参考 |
| YOLO11n-960 接近本文模型 | 强调 P2/CA 增益有限，论文转向系统实验分析；必要时继续优化结构 |
| FPS 下降明显 | 讨论精度-速度折中，并增加轻量部署场景限制 |
| 期刊认为创新不足 | 增强方法命名、结构图、小目标专项分析和公平对比矩阵 |
| 官方 VisDrone 平台不可用 | 明确只报告验证集，避免伪造官方 AP |
