# 《计算机工程与应用》投稿达标执行总计划

本文档作为后续工作的主控计划。目标是把当前 YOLO-VisDrone 项目推进到可以正式投向《计算机工程与应用》的完整中刊稿件状态：实验矩阵完整、论文主张稳健、数据证据可追溯、图表规范、代码与复现流程清楚、导师和审稿人可能追问的问题都有材料支撑。

需要说明的是，期刊录用结果受审稿意见、同期稿件、选题偏好和编辑部流程影响，不能由项目本身保证。因此本文档中的“100%”定义为可控范围内的投稿达标：凡是作者能够准备、验证和修正的内容，都按中刊投稿标准做到尽可能完整，不留下明显硬伤。

## 1. 投稿目标

- 目标期刊：《计算机工程与应用》。
- 论文类型：面向无人机航拍小目标检测的轻量化 YOLO 改进与系统实验分析论文。
- 核心任务：围绕 VisDrone2019-DET 场景中小目标多、密集遮挡、类别相似和复杂背景等问题，验证基于 YOLO11n 的 P2 高分辨率检测分支、CoordAttention 位置注意力、高分辨率输入和训练策略对检测性能、速度和复杂度的影响。
- 投稿达标标准：论文篇幅、相关工作、实验对比、消融分析、可视化、参考文献、复现材料和 GitHub 展示均达到中文期刊稿件应有完整度。

## 2. 论文主线

建议将论文主线固定为：

> 针对无人机航拍图像中小目标占比高、目标分布密集、遮挡严重和类别外观相似等问题，构建一种融合 P2 高分辨率检测分支、CoordAttention 位置敏感注意力和高分辨率输入策略的轻量化 YOLO11n 检测方法，并通过公平分辨率对照、主流 YOLO 基线、消融实验、尺度分组统计、速度复杂度分析和可视化案例系统验证其有效性。

暂定方法名称：

- 英文简称：`HRPCA-YOLO11n`
- 中文名称：融合高分辨率 P2 分支与坐标注意力的轻量化 YOLO11n 无人机小目标检测方法

最终名称需要根据全部公平对照实验结果微调。如果后续结果显示结构改进优势有限，论文主张应转为“轻量化模型在精度、速度和复杂度之间的系统折中分析”，避免夸大。

## 3. 当前基础

已完成并可作为论文基础的材料包括：

- VisDrone2019-DET 数据已整理为 YOLO 格式，训练、验证、推理流程可运行。
- 已完成 YOLO11n、YOLO11n-P2、YOLO11n-P2-CA、YOLO11n-P2-CA-960、YOLO11n-P2-CA-SmallObjAug 等主线实验。
- 已完成 YOLOv8n-640、YOLO11s-640 等外部基线初步对照。
- 已整理主结果、消融结果、模型复杂度、速度结果、类别级结果、目标尺度分布和尺度分组匹配统计。
- 已生成训练曲线、PR 曲线、混淆矩阵、检测可视化、失败案例、尺度分析图和精度-速度折中图。
- 已有 LaTeX/PDF 投稿候选稿、Markdown 草稿、README、复现实验命令和证据审计脚本。

当前不能直接作为最终中刊稿的原因：

- 960 输入带来的收益尚未通过 YOLO11n-960、YOLO11n-P2-960、YOLOv8n-960、YOLO11s-960 完全隔离。
- 外部主流 YOLO 对照还不够充分，不能只拿 YOLOv8n-640 与 YOLO11n-P2-CA-960 做结论性比较。
- 当前论文体量和论证深度更接近短论文，需要扩展为期刊长文。
- 参考文献数量、近三年相关工作覆盖和中文期刊写法还需要加强。
- 论文中的所有旧草稿、HTML 预览和表格必须统一到最新证据，不能保留过期速度数值或不一致表述。

## 5. 证据规则

1. 论文中所有数值必须来自真实运行日志、`runs/`、`paper/tables/`、脚本输出或官方评测返回结果。
2. 未完成 100 epoch 的服务器实验不得进入论文主结果表。
3. 服务器中间训练结果只用于进度判断，不用于论文结论。
4. 文献中的指标只能作为“文献报告结果”，不能与本项目复现实验混为同一协议公平对比。
5. VisDrone test-dev 官方结果只有在平台真实返回后才能写入正文。
6. README 面向教授和读者介绍项目，不写内部进度汇报、不写上传范围、不写未完成事项。
7. 旧 Markdown/HTML 草稿中的陈旧速度或结论必须清理或标注为历史草稿，正式论文以 LaTeX、PDF 和 `paper/tables/` 为准。

## 7. 风险预案
